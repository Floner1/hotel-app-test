Refactor and Codebase Optimization: Schema syncing, ORM tuning, and Template DRYing

Database Schema Synchronization:

Context Processors Migration:
- Extracted localized global data mappings (_CONTENT_DEFAULTS, HotelServices) entirely from primary views.

Test Infrastructure Improvements:
- Updated testing configuration (pytest.ini) adding '--nomigrations' flags, ensuring seamless compatibility over locally unmanaged T-SQL architectures.

Frontend Componentization Strategy (DRY pattern applied):
- Stripped historically duplicated framework HTML wrappers (DOCTYPE, headers) from core application views mapping back to admin_reservations.html, manage_accounts.html, and room_dashboard.html.
